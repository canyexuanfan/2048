# SSL证书域名不匹配问题排查指南

## 问题描述
- **错误信息**: `NET::ERR_CERT_COMMON_NAME_INVALID`
- **现象**: HTTPS访问时浏览器提示"你的连接不是专用连接"，证书通用名称与域名不匹配
- **域名**: `2048.hnwen17.top`
- **服务器**: 腾讯云 + 宝塔面板

## 已尝试的修复方法及失败原因

### ❌ 尝试1: 初步分析（2025-10-15 22:56）
- **方法**: 检查DNS解析和端口连通性
- **结果**: DNS解析正常，80/443端口通畅，HTTP可访问
- **失败原因**: 问题不在解析或连通性，而是SSL证书配置问题

### ✅ 尝试2: 证书信息检查（2025-10-15 22:58）
- **方法**: 使用openssl检查当前证书信息
- **发现**: 当前证书的CN（Common Name）为 `hnwen17.top`，不包含子域名 `2048.hnwen17.top`
- **根本原因确认**: 子域名使用了主域名的证书，导致域名不匹配错误

## 深层问题分析

### 可能的根本原因
1. **证书域名不匹配**: 当前SSL证书可能是为主域名 `hnwen17.top` 或其他域名申请的，不包含子域名 `2048.hnwen17.top`
2. **通配符证书缺失**: 如果使用通配符证书 `*.hnwen17.top`，可能配置不正确
3. **多域名证书配置错误**: SAN (Subject Alternative Names) 中未包含目标子域名
4. **证书绑定错误**: 宝塔面板中可能绑定了错误的证书到该站点

### 需要验证的关键点
- 当前证书的Common Name和SAN列表
- 宝塔面板中该站点绑定的具体证书
- 证书是否为该子域名专门申请

## 下一步排查策略

### 🎯 立即修复方案（已确认问题）
**问题**: 子域名 `2048.hnwen17.top` 使用了主域名 `hnwen17.top` 的证书

**解决步骤**:
1. **宝塔面板操作**:
   - 登录宝塔面板
   - 网站 → 选择 `2048.hnwen17.top` 站点 → 设置 → SSL
   - 点击 "Let's Encrypt" 标签
   - 在域名框中输入: `2048.hnwen17.top`
   - 点击 "申请" 按钮
   - 等待申请成功后，开启 "强制HTTPS"

2. **验证修复**:
   - 浏览器访问 `https://2048.hnwen17.top`
   - 应该不再出现证书错误

### 备选方案
如果单独申请失败，可以申请通配符证书:
- 为主域名申请通配符证书 `*.hnwen17.top`
- 该证书可同时覆盖所有子域名

## 注意事项
1. **域名验证**: Let's Encrypt申请时需要确保域名解析正确
2. **证书更新**: 申请新证书后需要重启Nginx服务
3. **缓存清理**: 浏览器可能缓存旧证书，需要清除缓存或使用隐私模式测试
4. **防火墙**: 确保443端口在腾讯云安全组中开放

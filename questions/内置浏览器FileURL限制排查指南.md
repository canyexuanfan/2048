# 内置浏览器 File URL 限制排查指南

## 问题描述

在 Codex 内置浏览器中尝试打开 `file:///F:/Trae%20AI/2048/index.html` 做本地页面预览时，浏览器安全策略拒绝访问该 `file://` 地址。

## 已尝试的修复方法及失败原因

- ❌ 直接用内置浏览器打开本地 `file://` 页面：被 Browser Use URL policy 拒绝。该限制属于浏览器安全策略，不能通过绕路、底层命令或替代浏览器控制方式规避。

## 深层问题分析

该问题不是 2048 项目代码错误，而是自动化浏览器对本地文件 URL 的访问限制。继续重复打开同一类 `file://` 地址只会浪费排查时间。

## 下一步排查策略

优先使用不依赖被拦截浏览器动作的验证方式，例如：

- `node --check` 检查 JavaScript 语法。
- `git diff --check` 检查补丁空白问题。
- 使用轻量 DOM mock 验证初始化、主题切换和移动逻辑。
- 如确需真实视觉预览，后续应由用户手动打开页面，或在用户确认后使用明确允许的本地开发服务器路线。

## 调试工具

- `node --check script.js`
- `git diff --check`
- 本地 Node 逻辑验证脚本

## 注意事项

不要把浏览器安全策略拦截误判为页面加载失败。不要尝试用其他浏览器自动化方式绕过该策略。

## 更新记录

- ✅ 2026-07-07：确认内置浏览器无法自动打开该本地 `file://` 页面，改用本地静态和逻辑验证继续推进。

# 2048.hnwen17.top 在腾讯云 + 宝塔的部署与排查（专用版）

> 域名：`2048.hnwen17.top` · 主域：`hnwen17.top` · 服务器公网IP：`121.4.65.95` · 面板：宝塔 Linux

## 一、域名解析（必须）

在腾讯云「云解析 DNS」为 `hnwen17.top` 添加 A 记录：

```
主机记录：2048
记录类型：A
记录值：121.4.65.95
TTL：600（默认即可）
```

验证（Windows PowerShell/命令行）：

```bash
nslookup 2048.hnwen17.top
ping 2048.hnwen17.top
```

若返回 `121.4.65.95` 即解析生效；若不生效，等待 5–30 分钟或刷新 DNS：

```bash
ipconfig /flushdns
```

## 二、宝塔面板创建站点（纯静态）

1) 登录面板：`http://121.4.65.95:8888`

2) 菜单「网站」→「添加站点」：

```
域名：2048.hnwen17.top
根目录：/www/wwwroot/2048.hnwen17.top
FTP：不创建
数据库：不创建
PHP：选择纯静态
```

3) 上传文件到根目录 `/www/wwwroot/2048.hnwen17.top`：

```
index.html
script.js
style.css
```

（删除默认 index.html/404.html 等占位文件；无需上传 server.py）

4) 配置文件（可选优化）：在站点「设置 → 配置文件」参考 `nginx_2048.conf.example` 内容。

## 三、端口与安全组（经常被忽略）

确保 80/443 已开放：

- 腾讯云实例 → 安全组 → 入站规则：`TCP 80/443/8888` 允许 `0.0.0.0/0`
- 宝塔面板 → 安全 → 放行端口：`80`、`443`（必要）、`8888`（面板，仅管理时需要）

测试 80 端口是否通：在浏览器访问 `http://121.4.65.95` 是否有页面响应（任意页面即可）。

## 四、启用 HTTPS（推荐）

站点「设置 → SSL → Let's Encrypt」：选择域名并申请，成功后启用「强制 HTTPS」。

## 五、快速自检清单

- [ ] `nslookup 2048.hnwen17.top` 输出 IP 为 `121.4.65.95`
- [ ] 能打开 `http://121.4.65.95`（端口通）
- [ ] 宝塔网站列表存在 `2048.hnwen17.top` 且运行中
- [ ] `/www/wwwroot/2048.hnwen17.top` 仅有 `index.html/script.js/style.css`
- [ ] 权限为 755，`index.html` 存在
- [ ] 配置了 SSL 并可访问 `https://2048.hnwen17.top`

## 六、定位失败的关键日志位置

在宝塔站点「设置 → 日志」或直接查看：

```
/www/wwwlogs/2048.hnwen17.top_access.log
/www/wwwlogs/2048.hnwen17.top_error.log
```

常见错误指引：

- 403：文件权限或缺少 `index.html`
- 404：`root` 路径或文件位置不对
- 502：选择了 PHP 但本项目为纯静态 → 改为纯静态

## 七、常见误区与修复

1) 仅配置了主域 `hnwen17.top`，未添加子域 `2048` 的 A 记录 → 按「一、域名解析」添加。

2) 文件上传在了主站目录 `/www/wwwroot/hnwen17.top` → 移动到子站目录 `/www/wwwroot/2048.hnwen17.top`。

3) 未开放 80/443 安全组规则 → 按「三、端口与安全组」放行。

4) 证书申请失败 → 确保 80 可访问；待解析完全生效再申请；重试时可更换邮箱或清理历史失败条目。
